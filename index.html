Notes

Items marked with * have some additional notes, which you can display just by hovering on them.

  • [1]: The Keccak hash function has won the SHA-3 competition and is therefore the underlying hash function in SHA-3. However, some modifications has been introduced in the final stage of standardization, which lead to two almost the same but not really hash functions. Keccak: as introduced by its authors and SHA-3: as standardized in FIPS-202. Many cryptocurrencies (such as Ethereum) chose SHA-3 for their hashing algorithm, but did so just before the standard was finalised. This resulted in using the Keccak functions instead of what is now considered as SHA-3.
    Source: Wikipedia, Ethereum SE, Ethereum yellow paper
  • [2]: EdDSA hashes the message internally before signing and this function can be chosen. The default is SHA-512. Some cryptocurrencies are leaving this upto the EdDSA algorithm exclusively and do not hash the message beforehand. Some do. This table tries to list both, the "outter" hash that is used before inserting it into the algorithm and also the EdDSA's internal one.
    Source: RFC 8032
  • [3]: Monero transactions are based on Elliptic Curve Cryptography using curve Ed25519, but transaction inputs are signed with so-called Multilayered Linkable Spontaneous Anonymous Group signatures (MLSAG), and output amounts (communicated to recipients via ECDH) are concealed with Pedersen commitments and Borromean ring signatures (later replaced with Bulletproofs).
    Source: From Zero To Monero, Ring confidential transactions, Confidential transactions, Borromean ring signatures, Bulletproofs
  • [4]: Vulnerabilities were found in the Curl hash function, causing several dispute between IOTA engineers and cryptographers.
    Source: Cryptographic vulnerabilities in IOTA, IOTA Vulnerability Report

Columns description

  • Type Describes if the cryptocurrency is based on the UTXO model or not. Transactions in the UTXO model always spend the whole input, with some of the coins returned as a change. In account account-based model, there is no such mechanism and just a simple account book is used.
  • Signing algorithm What signing algorithm is used, all present coins use Elliptic Curve Cryptography.
  • Curve What elliptic curve is used in the underlying signing algorithm.
  • Hash What hash function is used to hash the transaction data, which are then signed.
  • Address encoding What algorithm is used to encode the address.
  • Address hash What hash function is used for addresses.
